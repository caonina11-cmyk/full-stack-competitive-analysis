# 分析框架

本文件收纳全栈竞品分析常用框架。使用时不要机械套模板，要根据行业、用户问题和输出目标取舍。

## 1. 全栈分析地图

全栈竞品分析的主链路：

```text
行业环境 → 用户与场景 → 商业模式 → 产品定位 → 功能体系 → 流程机制 → 核心路径 → 体验与界面 → 增长与转化 → 战略信号 → 产品与体验决策 → 我方行动
```

每一层都要沉淀为判断：

| 层级 | 核心问题 | 产出 |
|---|---|---|
| 行业环境 | 为什么这个赛道现在重要？ | 趋势、约束、竞争焦点 |
| 用户与场景 | 谁的问题最值得解决？ | 用户角色、任务、场景、痛点 |
| 商业模式 | 公司靠什么赚钱和增长？ | 收入逻辑、定价、渠道、壁垒 |
| 产品定位 | 它想占据什么心智？ | 定位差异、价值主张 |
| 功能体系 | 它用哪些能力承接定位？ | 产品矩阵、KANO、功能差距 |
| 流程机制 | 功能如何在用户流程里推动下一步？ | 阶段、阻力、机制、指标影响、功能机会 |
| 核心路径 | 用户如何完成关键任务？ | 路径图、摩擦点、亮点 |
| 体验与界面 | 它如何让用户理解和信任？ | 信息架构、交互、文案、视觉 |
| 增长与转化 | 设计如何服务商业目标？ | 获客、转化、留存机制 |
| 战略信号 | 它接下来可能往哪里走？ | 招聘、专利、资本、组织、迭代 |
| 产品与体验决策 | 我们该如何取舍？ | 方向、用户、功能、体验、表达、优先级 |

## 2. 七个商业与战略 Lens

用于建立公司、行业、商业模式和战略信号的结构化判断。

1. **财务与业务结果**：收入来源、成本结构、增长杠杆、利润压力、周期性风险。
2. **市场供给与商业模式**：目标市场、买方/用户/影响者/阻碍者、价值创造和价值捕获方式。
3. **产品组合与产品结果**：产品线、服务、平台、生态、数字化与人工服务边界。
4. **竞争动态**：直接竞品、间接竞品、替代方案、新进入者，差异化在哪里赢或输。
5. **趋势与战略担忧**：监管、AI、自动化、买方预期、行业整合、商品化风险。
6. **产品研发与体验方式推断**：产品驱动、销售驱动、服务驱动；平台化程度；发现、实验、数据和 AI 成熟度。推断必须标注依据。
7. **战略信号**：专利、招聘、领导层变化、组织调整、资本动作和产品更新。

不要写成百科。每个 lens 都要回答“这对产品方向、功能策略、体验路径和界面表达有什么影响”。

## 3. PEST/PESTEL

用于宏观环境分析：

- Political/Policy：政策、监管、政府扶持或限制
- Economic：市场规模、增长率、资本环境、成本变化
- Social：用户习惯、人口结构、文化观念、人才供给
- Technological：AI、自动化、基础设施、技术成熟度
- Environmental：可持续、能耗、环保约束，按行业选用
- Legal：合规、数据安全、隐私、知识产权，按行业选用

重点不是罗列新闻，而是说明它如何改变竞争焦点和产品设计约束。

## 4. 商业模式画布

用 9 要素描述公司如何创造、传递和获取价值：

| 要素 | 核心问题 |
|---|---|
| 客户细分 | 为谁服务？最重要客户是谁？ |
| 价值主张 | 解决什么问题？提供什么独特价值？ |
| 渠道通路 | 如何触达客户？ |
| 客户关系 | 自助、销售、服务、社区还是伙伴？ |
| 收入来源 | 靠什么赚钱？用户为什么付费？ |
| 核心资源 | 数据、品牌、技术、渠道、内容、供应链还是生态？ |
| 关键业务 | 必须持续做好的活动是什么？ |
| 重要伙伴 | 上下游、渠道、集成、生态伙伴是谁？ |
| 成本结构 | 最大成本项和规模化约束是什么？ |

竞品分析要特别关注：商业模式如何影响注册路径、定价页、功能开放策略、权限体系、试用体验和销售触点。

## 5. 用户与场景分析

区分：

- 买方：谁付钱或审批
- 使用者：谁每天使用
- 决策者：谁决定采购或采用
- 影响者：谁影响选择
- 管理者：谁配置、审计、管理
- 阻碍者：谁可能反对或拖慢采用

场景分析至少回答：

- 用户要完成什么任务
- 触发时刻是什么
- 成功标准是什么
- 失败代价是什么
- 当前替代方案是什么
- 哪些信息会建立信任
- 哪些步骤会造成放弃

## 6. KANO 功能分类

用于功能对比和优先级判断：

- **基本型**：没有就不合格，有了也不惊喜。决定能否进入牌桌。
- **期望型**：做得越好越满意。决定竞争力。
- **兴奋型**：用户未预期但能制造差异。决定记忆点和传播点。

输出时不要只写“有/无”。要说明：

- 实现深度
- 使用路径
- 用户价值
- 是否影响转化、留存、效率或信任
- 我方是否值得跟进

## 7. UX 五层模型

从战略到表现拆解产品：

1. 战略层：产品目标和用户需求
2. 范围层：功能规格和内容需求
3. 结构层：交互设计和信息架构
4. 框架层：界面布局、导航、信息设计
5. 表现层：视觉风格、品牌感、动效和细节

用于判断“界面现象背后的产品意图”。例如，一个复杂定价页可能不是设计差，而是企业销售导向导致的线索收集优先。

## 8. 增长与转化分析

观察设计如何服务增长：

- 获客入口：SEO、内容、广告、社区、模板、免费工具、应用市场
- 激活：注册后多久看到价值，是否有示例数据、模板、引导
- 转化：定价页、试用限制、付费墙、联系销售、信用卡要求
- 留存：通知、协作、内容更新、数据沉淀、工作流绑定
- 扩张：团队邀请、权限、集成、用量阶梯、模板生态

把增长机制翻译成设计问题：入口是否清楚、价值是否快速显现、风险是否被降低、下一步是否明确。

## 9. 战略信号

战略信号通常比宣传更诚实：

| 信号 | 看什么 | 可能说明什么 |
|---|---|---|
| 招聘 | 岗位数量、角色、技能、地点、资深程度 | 投入方向、组织能力、技术栈 |
| 专利 | 技术主题、申请时间、发明人团队 | 研发赌注、技术壁垒 |
| 领导层 | 新 CTO/CPO/CDO/设计负责人、来源 | 战略转型、组织补课 |
| 产品更新 | 更新频率、重点模块、废弃功能 | 路线图重心 |
| 资本动作 | 融资、并购、拆分、上市 | 扩张方式和压力 |
| 伙伴生态 | 集成、渠道、开发者、代理商 | 平台化或分销策略 |

所有战略信号都要区分事实和推断。

## 10. SWOT 与行动建议

SWOT 只在能导向行动时使用：

- Strengths：我方相对优势
- Weaknesses：我方相对短板
- Opportunities：外部机会
- Threats：外部威胁

建议使用交叉策略：

- SO：用优势抓机会
- ST：用优势应对威胁
- WO：补短板抓机会
- WT：规避弱点和威胁叠加

行动建议必须包含优先级、依据、影响对象和落地方式。
