# 产品体验与设计走查

本文件吸收 UX 竞品分析的核心，用于把“看界面”升级为“理解产品策略如何落到体验和设计”。

## 目录

1. 走查原则
2. 关键路径
3. 评估维度
4. 对比矩阵
5. 机会地图

## 1. 走查原则

- 分析完整旅程，不只看孤立页面。
- 关注 UX 质量，不只看功能是否存在。
- 同时看用户目标和商业目标。
- 记录事实、推断意图、判断影响、给出行动。
- 可以加入相邻领域标杆，但要标注“标杆参考”，不要混同为直接竞品。

## 2. 关键路径

根据产品类型选择路径，优先比较同一用户任务。

### 官网/营销站路径

- 首页 → 核心价值理解
- 首页 → 产品/功能页
- 首页 → 定价页
- 首页 → 注册/试用/预约演示
- 内容页/案例页 → 转化入口

### SaaS/工具路径

- 注册 → onboarding → 首次价值
- 创建项目/导入数据/选择模板
- 完成核心任务
- 分享/协作/导出
- 触发付费/升级/邀请团队

### 电商/交易路径

- 搜索/浏览 → 商品/服务详情
- 对比/筛选 → 加购/预约
- 下单/支付 → 售后/复购

### 内容/社区路径

- 发现内容 → 消费内容 → 互动/关注
- 发布内容 → 获得反馈 → 复访
- 社区身份、激励和治理机制

## 3. 评估维度

| 维度 | 观察问题 |
|---|---|
| 信息架构 | 用户能否快速理解产品结构和下一步？导航层级是否清晰？ |
| 交互路径 | 核心任务步骤数、分支、反馈、错误处理是否合理？ |
| 功能可见性 | 重要能力是否被看见、理解、试用？ |
| 内容策略 | 文案是否说清对象、场景、价值、证据和风险？ |
| 视觉表达 | 视觉是否承载定位、信任、效率或情绪？是否只是装饰？ |
| 转化设计 | CTA、定价、试用、销售线索是否自然？阻力在哪里？ |
| 移动体验 | 响应式、触控、长页面阅读、表单输入是否顺畅？ |
| 可访问性 | 对比度、字号、键盘、语义、错误提示是否友好？ |
| 性能体感 | 首屏、加载、动效、等待反馈是否影响信任和效率？ |
| 空/错/边界状态 | 新用户、无数据、失败、权限不足时是否有清晰引导？ |

## 4. 对比矩阵

用于多品对比：

| 关键任务 | 竞品 | 支持程度 | 步骤数 | UX 质量 1-5 | 亮点 | 摩擦点 | 背后意图 | 我方启示 |
|---|---|---:|---:|---:|---|---|---|---|
| 注册并看到首次价值 | A | 完整 | 4 | 4 | 模板引导强 | 需要邮箱验证 | 降低激活门槛 | 可借鉴模板式 onboarding |

评分建议：

- 1：不可用或严重阻碍任务
- 2：可完成但摩擦明显
- 3：行业平均，可用但缺乏亮点
- 4：顺畅且有清晰设计意图
- 5：显著优于同类，能改变用户行为或转化结果

## 5. 机会地图

把体验观察转化为产品设计机会：

| 机会类型 | 来源 | 说明 | 判断 |
|---|---|---|---|
| 行业标配补齐 | 多数竞品已有 | 缺失会影响基本信任 | 需要做，但不应过度投入 |
| 差异化机会 | 少数竞品做得好 | 可形成记忆点或效率优势 | 值得设计探索 |
| 反向定位 | 竞品普遍复杂/昂贵/重销售 | 用简单、自助、透明形成区隔 | 适合切入细分用户 |
| 体验降阻 | 路径步骤多或信息不清 | 降低理解、注册、配置、购买成本 | 通常是高 ROI 改进 |
| 表达升级 | 价值说不清或证据不足 | 用结构、文案、案例、视觉建立信任 | 影响官网和产品内体验 |

最终不要只写“竞品 A 做得好”。要写清楚：

```text
它做了什么 → 可能为什么这么做 → 对用户/商业有什么影响 → 我们学什么/不学什么/怎么改
```
